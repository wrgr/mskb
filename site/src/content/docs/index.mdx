## Inside a concept page

Every concept page has the same structure — designed so you always know where you are and what to read next:

- **Learning objectives** — what you should be able to do, tagged by Bloom's level
- **Prerequisites** — concepts you should understand first
- **Key questions** — self-check questions to test your understanding
- **Papers** — 3–7 foundational references drawn from the citation graph
- **Resources** — videos, tools, and datasets curated against a quality rubric

The citation graph answers *"what's the research landscape?"*; the concept map answers *"what should I learn next?"*; resources answer *"how do I learn it?"* They're separate on purpose.

## About the corpus

MSKB is built from an OpenAlex-derived bibliometric pipeline (retrieve → score → cluster → distill) layered with a hand-curated concept ontology. The corpus starts from **40 hand-curated seed papers** across {topicCount} MS research topics, expanded via one-hop citation networks and filtered by cross-seed connectivity, within-subdomain citation scores, and expert-signal curation. The current release ({siteStats.asOf}) includes **{siteStats.curatedPapers} core papers** and their citation neighborhoods — **{siteStats.totalArtifacts.toLocaleString()} total artifacts** in the network — linked to **{conceptCount} learner-centered concept pages**. Every design decision is documented.

<Aside type="note" title="AI-assisted curation — read this first">
MSKB is a **learning engineering experiment** built to support student access to MS research. The corpus was assembled and annotated with AI assistance (bibliometric pipeline + Claude API distillation). Plain-language summaries are AI-generated from the primary literature. Verify critical clinical claims against primary sources and current guidelines before applying them.
