## What every concept page shows you

Every concept page has the same structure:

- **Learning objectives** — what you should be able to do, tagged with Bloom's taxonomy level (Remember → Understand → Apply → Analyze)
- **Prerequisites** — concepts you should understand first
- **Key questions** — self-check questions to test your understanding
- **Papers** — 3–7 foundational references drawn from the citation graph
- **Resources** — videos, tools, and datasets curated against a quality rubric

Concepts, citations, and resources are kept **separate** on purpose: the citation graph answers *"what's the research landscape?"*; the concept map answers *"what should I learn next?"*; resources answer *"how do I learn it?"*

## About the corpus

MSKB is built from an OpenAlex-derived bibliometric pipeline (retrieve → score → cluster → distill) layered with a hand-curated concept ontology. The corpus starts from **40 hand-curated seed papers** across 18 MS research topics, expanded via one-hop citation networks and filtered by cross-seed connectivity, within-subdomain citation scores, and expert-signal curation. Every design decision is documented.

<Aside type="note" title="AI-assisted curation — read this first">
MSKB is a **learning engineering experiment** built to support student access to MS research. The corpus was assembled and annotated with AI assistance (bibliometric pipeline + Claude API distillation). Plain-language summaries are AI-generated from the primary literature. Verify critical clinical claims against primary sources and current guidelines before applying them.
